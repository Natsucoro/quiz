# 📂 プロジェクト・ファイル構成図

開発チーム（システム・コンテンツ・デザイン）が円滑に連携し、各担当が独立して作業を進められるように設計した構成です。

```text
quiz/
├── .github/                        # CI/CD設定 (GitHub Actions)
│
├── docs/                           # ドキュメント類
│   ├── 設計書（Ver. 1.8）.md
│   └── ファイル構成.md
│
├── frontend/                       # 【デザイン調整担当 & システム担当】React(TypeScript)
│   ├── public/
│   │   ├── icons/                  # ジャンル別フラットアイコン（商用無料）
│   │   ├── se/                     # 効果音（mp3/ogg形式で軽量化）
│   │   ├── manifest.json           # PWA設定
│   │   └── sw.js                   # Service Worker（オフラインキャッシュ）
│   ├── src/
│   │   ├── components/
│   │   │   ├── TopPage/            # ジャンル選択・難易度選択（表示切替）
│   │   │   ├── GamePage/           # クイズ本編・結果発表（表示切替）
│   │   │   └── common/             # 常設アイコン・トースト通知・吹き出し
│   │   │       ├── Settings/       # 設定コンポーネント（全画面から呼び出し可）
│   │   │       └── Login/          # ログインモーダル（設定画面のログインボタンから表示）
│   │   ├── services/
│   │   │   ├── speechRecognition.ts  # Web Speech API（音声認識）
│   │   │   ├── speechSynthesis.ts    # Web Speech API（読み上げ）
│   │   │   ├── quizEngine.ts         # 出題ロジック・既読管理
│   │   │   └── payment.ts            # Stripe決済処理
│   │   ├── store/                    # アプリ全体の状態管理（Zustand）
│   │   │   ├── settingsStore.ts      # 音量・読み上げ速度・モード
│   │   │   └── purchaseStore.ts      # 課金フラグ・購入履歴
│   │   ├── hooks/
│   │   │   ├── useHandsFree.ts       # ハンズフリーモード管理
│   │   │   └── useOffline.ts         # オフライン状態検知
│   │   ├── styles/                   # フラットデザイン定義（配色・CSS）
│   │   ├── App.tsx
│   │   └── main.tsx
│   ├── package.json
│   └── tsconfig.json
│
├── backend/                        # 【システム開発担当】Cloud Run (Python/FastAPI)
│   ├── api/                        # エンドポイント（/quiz, /payment, /user）
│   ├── core/                       # タイマー・正誤ループロジック
│   ├── models/                     # Pydanticデータ定義（問題・ユーザー）
│   ├── services/
│   │   ├── firestore.py            # Firestore操作（有料版既読管理）
│   │   ├── stripe.py               # Stripe決済処理（PayPay/カード）
│   │   └── logger.py               # 音声認識エラーログ（Firestoreに記録）

│   ├── Dockerfile                  # Cloud Run デプロイ用

├── data/                           # 【クイズ制作担当】
│   └── quizzes/                    # ジャンル別・レベル別クイズデータ（JSON）
│       ├── animals.json            # 動物
```

## 各担当の作業範囲

| 担当 | 作業ディレクトリ |
| :--- | :--- |
| クイズ制作 | `data/quizzes/` |
| システム開発 | `frontend/src/services/`, `frontend/src/hooks/`, `frontend/src/store/`, `backend/` |
| デザイン調整 | `frontend/src/components/`, `frontend/src/styles/`, `frontend/public/` |
| AutoGen開発 | `scripts/quiz_dev.py` |

## 設計上のポイント

| 項目 | 内容 |
| :--- | :--- |
| 状態管理 | Zustandで設定・課金フラグを一元管理。コンポーネント間のバケツリレーを防ぐ |
| 音声認識ログ | 認識失敗・誤認識をFirestoreに記録。後からクイズ精度を改善するためのデータ基盤 |
| 効果音 | mp3/ogg形式で軽量化。初期読み込みの速さがサクサク感に直結 |
