# 📂 プロジェクト・ファイル構成図

開発チーム（システム・コンテンツ・デザイン）が円滑に連携し、各担当が独立して作業を進められるように設計した構成です。

```text
my_ai_team/
├── .github/                # CI/CD設定 (GitHub Actions)
├── docs/                   # 仕様書・設計書等のドキュメント
├── src/
│   ├── frontend/           # 【デザイン調整担当 & システム担当】
│   │   ├── public/         # 静的資産（アイコン、SE：ピンポン音等）
│   │   ├── src/
│   │   │   ├── components/ # クイズ表示画面、コンシェルジュ吹き出し演出
│   │   │   ├── services/   # 音声認識(Web Speech API)、APIクライアント
│   │   │   └── styles/     # Toy Design定義（配色、ぷっくり感のCSS）
│   │   └── package.json
│   │
│   ├── backend/            # 【システム開発担当】
│   │   ├── api/            # 各エンドポイント（/quiz, /payment, /user）
│   │   ├── core/           # コンシェルジュの知能（タイマー、正誤ループ）
│   │   ├── models/         # Pydanticデータ定義（問題・ユーザー）
│   │   ├── services/       # Stripe決済処理（150円設定）、Firestore操作
│   │   └── main.py         # FastAPIエントリーポイント
│   │
│   └── data/               # 【クイズ制作担当】
│       └── quizzes/        # ジャンル別・レベル別のクイズデータ（JSON）
│           ├── animals.json
│           ├── history.json
│           └── vehicle.json
│
├── scripts/                # クイズデータのバリデーション・一括登録ツール
├── tests/                  # 単体テスト・結合テストコード
├── Dockerfile              # Google Cloud Run デプロイ用構成
├── .env.development        # 開発用環境変数（Stripeテスト鍵等）
├── .env.production         # 本番用環境変数（Stripe本番鍵等）
├── .gitignore              # .env等の機密情報を除外
├── README.md               # プロジェクト全体の概要・起動手順
└── requirements.txt        # Python依存ライブラリ一覧
