# 📱 わたしはダレでしょう？クイズ集！設計書（Ver. 1.8）

## 1. プロジェクト概要
* **コンセプト**: 運転中の安全と「知的な発見」を両立する、音声操作特化型の学習エンタメアプリ。
* **ミッション**: クイズを通じ、遊びながら自然に「物知り博士」になれる体験を提供。
* **デザイン言語**: フラットデザイン統一、ふわふわしたアニメーション、ほんわかな配色。
* **キャラクター性**: **「コンシェルジュ」**。丁寧かつ簡潔な案内を行い、ユーザーの思考を妨げない。

---

## 2. ユーザー体験（UX）とナビゲーション
### 2.1 ナビゲーションフロー
1. **TOP（ジャンル選択）**: タップで選択（全8ジャンル）。右上に常設アイコン＋ログイン状態表示（未ログイン時は「ゲスト」と表示）。
2. **難易度選択**: Lv. 1〜10（5以下は「こども」、6以上は「おとな」）。
3. **問題数**: 常に10問固定（選択なし）。
4. **クイズ本編**: コンシェルジュによる出題 ➔ 正解・降参で次問へ。
5. **結果発表**: スコア表示 ➔ 次の操作へのガイド。

### 2.2 設定・オプション
* **デザイン**: フラットデザイン統一（アイコン・UI全体）。商用無料のフラットアイコンを使用。
* **読み上げ速度**: 0.5〜2.0倍の範囲で変更・保存可能（localStorage）。
* **常設アイコン（右上に4つ配置）**:
  * ⚙️ 設定（ログインはここから）
  * 🔊/🔇 音声（初期状態はミュート）
  * 🤲 ハンズフリーモード（ON：色あり / OFF：グレーアウト）
  * 🏠 TOP（確認ダイアログあり：「クイズを中断しますか？スコアはリセットされます」）
* **ログイン状態表示**: 未ログイン時は右上に「ゲスト」と表示。ログイン時は、ログイン済と表示。タップでメールアドレスを表示。また、ログアウト可能に。
* **テスト用アカウント**: ログインアカウント'test@example.com' 入力ですべてを「購入済み」として扱う。

### 2.3 結果発表画面
* 正解数 / 総問題数（例：7/10問正解）
* 正解率（%）
* 間違えた問題の一覧（復習用）
* 次のアクションボタン：
  * もう一度同じジャンル・同じレベルで遊ぶ
  * 別のレベルへ
  * 別のジャンルへ
  * TOPに戻る

## 4. クイズエンジン・操作ロジック
### 4.0 クイズ本編の画面表示要素
* 選択中のジャンル名 ＋ ジャンルアイコン
* 問題文（大きく中央表示）
* ヒントボタン（タップで内容を表示）
  * 通常モード：ヒント1のみ
  * ハンズフリーモード：ヒント1・2・3
* 降参ボタン
* 残り問題数（例：3/10）
* 現在のスコア（正解数）
* ハンズフリーモード時：吹き出し（「〇〇と言ってみてね」）
* 常設アイコン4つ（右上）

### 4.1 フィードバックと遷移
| イベント | ハンズフリーモード | 通常モード |
| :--- | :--- | :--- |
| **正解時** | 🔔（ピンポン）➔ ●アイコン表示 ➔「答えは〇〇です。次の問題です。」➔ 次問へ | ●アイコン表示 ➔「正解は〇〇です。」と表示 ➔ 次問へ |
| **不正解時** | 🎵（短いSE）➔ ×アイコン表示 ➔ 次の発話を待機（再挑戦） | ×アイコン表示 ➔ 再挑戦（同じ4択のまま） |
| **降参時** | （確認なし）➔ ●アイコン表示 ➔「答えは〇〇です。次の問題です。」➔ 次問へ | ●アイコン表示 ➔「正解は〇〇です。」と表示 ➔ 次問へ |
* **ループ制**: 正解または降参がない限り、次の問題へは進まない（両モード共通）。

### 4.2 音声コマンド（日本語のみ対応）
* **「問題」**: 現在のメイン問題を再読。
* **「ヒント1 / ヒント2 / ヒント3」**: 段階的なヒントの読み上げ。
* **降参コマンド（予約語3〜5種）**: 「降参」「答え」「わからない」など ➔ 正解を読み上げ次問へ。
* コマンドは部分一致・類似語も拾う（例：「わからん」→降参扱い）。

### 4.3 コンシェルジュの誘導（ハンズフリー時）
1.  **7秒沈黙**: 「ヒント1と言ってみてね」
2.  **12秒沈黙**: 「ヒント2と言ってみてね」
3.  **17秒沈黙**: 「ヒント3と言ってみてね」
4.  **25秒沈黙**: 「降参の場合は、降参と言ってね」

### 5.4 既読（出題済み）管理
* **管理単位**: ジャンル × 難易度ごと（またぎ管理は不要）。
* **1プレイ中**: 同じ問題は絶対に出ない。
* **プレイをまたぐ場合（有料版のみ）**: 出題済み問題はサーバ（Firestore）に保存し、次回プレイでも除外。
* **プレイをまたぐ場合（ゲスト・無料版）**: 既読管理は行わない。毎回プレイ開始時に全問からランダム出題。同じ問題が出ることがある。
* **自動リセット**: 対象ジャンル×難易度の全問を出題し終えたとき、次の問題を取得するタイミングで自動リセット。
* **手動リセット**: 設定画面から「既読をリセット」ボタンで任意にリセット可能（有料版のみ）。

### 5.5 ゲストの10問制限と既読管理の関係
* ゲストは各ジャンル×難易度ごとに **10問まで**プレイ可能（課金制限）。
* ただし「既読除外」は行わないため、**何度でも同じ10問セットで遊べる**。
* 「10問完了」のロック表示は出さない。ゲストは何度でも同じ難易度を遊べる。
* 購入（ステップ1）すると問題数が60問に増え、かつ既読管理が有効になる。

## 6. フリーミアム・課金設計
### 6.1 ゲスト（無料）の制限
* 全ジャンル選択可。
* 選択できる難易度：1・2・6・7のみ（こどもとおとな両方を体験させる設計）。
* 各ジャンル×難易度ごとに10問まで。

### 6.2 段階的購入（買い切り・ジャンル×難易度ごと）
| ステップ | 内容 | 価格 |
| :--- | :--- | :--- |
| 無料 | 10問 | 無料 |
| ステップ1 | +50問（計60問） | +120円 |
| ステップ2 | +50問（計110問） | +120円 |

### 6.3 難易度3・4・5・8・9・10の解放
* 無料では選択不可。
* ステップ1（+120円）で50問＋難易度解放。
* ステップ2（+120円）でさらに50問追加（計110問）。

### 6.4 課金UI
* ロックされた難易度・ステップは🔒アイコンで表示。
* 🔒をタップすると購入画面へ遷移。
* ロックアイコンの下に「120円で解放＆60問追加！」など小さく表示。

### 6.5 購入特典
* 「既読管理（正解済みを除外）」の有効化。

## 8. オフライン対応
* **PWA対応**: Service Workerで問題データ・正解データをキャッシュし、オフラインでもクイズが遊べる。
* **オフラインで使える機能**: クイズ本編・音声読み上げ・音声認識・既読管理（無料版）。
* **オフラインで使えない機能**: 課金・購入・既読管理（有料版/Firestore同期）。
* **課金・購入UIのオフライン時の挙動**:
  * ボタンは通常通り表示（グレーアウトなし）。
  * タップするとトースト通知「オフラインのため購入できません。接続後にお試しください。」を表示。

## 10. AI クイズ制作・検閲ガイドライン
### 10.1 【作成AI】データ作成ルール
*   **100% ジャンル限定**: `answer` および `dummy1〜3` に、設定されたジャンル以外の生物や物を絶対に入れない（例：哺乳類ジャンルなら鳥・魚・爬虫類・昆虫は禁止）。
*   **回答の重複禁止（レベル内）**: 1つのレベル（50問）の中で、同じ動物を二度正解にしない。
*   **レベル間の重複抑制**: 前レベルで正解になった動物は、次レベルの全50問中 5問（10%）以内 に抑える。
*   **音声回答の決定打（キラー・ヒント）**: 音声で聞いただけで「それしかない」と特定できる固有の特徴を必ず入れる（例：しかくい、うんち／7本の、ゆび等）。
*   **ターゲットに合わせた難易度**: 各レベルの対象年齢に合わせた語彙と動物の選定を行う。
*   **漢字とルビの仕様**: 学年に合わせた漢字を使い、`questionRuby` および `hint1Ruby` キーには必ず `{漢字|かんじ}` の形式を適用する。

### 10.2 【チェックAI】検閲ルール
*   **構造の完全一致チェック**: 指定された 13個のキー 以外（rationale や questionNumber など）が混入していないか確認し、あれば即排除する。
*   **生物分類・カテゴリの厳密チェック**: 選択肢の4種すべてが本当に指定されたジャンルに属しているか一頭（一つ）ずつ再確認する。
*   **重複スキャン**: 全50問の answer をリスト化し、重複がないか、前レベルとの被りが5問以内かをカウントする。
*   **ルビの構文エラー検出**: `{ | }` の閉じ忘れや、漢字以外の文字にルビが振られていないかを精査する。
*   **事実確認（ファクトチェック）**: 「世界一」「日本最大」などの事実が最新の生物学データと合致しているか確認する。
